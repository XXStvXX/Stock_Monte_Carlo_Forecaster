# AI-Assisted Workflow

This project uses AI as a development assistant, not as a source of financial truth.

AI-assisted work was useful for:

- comparing package and module-organization options
- drafting documentation and interview explanations
- generating implementation ideas for CLI, tests, and Streamlit UI iteration
- identifying assumptions and limitations that needed manual review
- improving wording for nontechnical readers

Human validation remains the core of the project:

- model assumptions are documented explicitly
- risk metrics are implemented in code and checked with tests
- outputs are interpreted as scenario analysis, not investment advice
- the author remains responsible for final code, claims, and explanations

The portfolio goal is to show a modern analyst/developer workflow:

```text
AI-assisted prototyping -> tested Python modules -> documented scenario-analysis app
```

AI helped speed up iteration, but the final reasoning, model limitations, and interview explanation are owned by the author.
